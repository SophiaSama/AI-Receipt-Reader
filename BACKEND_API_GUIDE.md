# Architecture-Aligned Backend Guide

The frontend has been refactored to align with the **Backend-Orchestrated AI** architecture. The client no longer performs OCR; it simply acts as a consumer of the results generated by the Lambda logic.

## Architecture Flow
`User App` -> `API Gateway` -> `Lambda` -> `(S3 + Mistral OCR + Mistral LLM + DynamoDB)`

## Required API Endpoints

### 1. Unified Process & Save
This is the primary endpoint for automated scanning.

- **Endpoint**: `POST /api/process`
- **Content-Type**: `multipart/form-data`
- **Request Body**: `file` (Binary Image)
- **Lambda Responsibility**:
  1. **Upload to S3**: Store raw image and get `imageUrl`.
  2. **Mistral AI OCR**: Send image to Mistral for text extraction.
  3. **Mistral AI LLM**: Send raw text to Mistral for JSON structuring.
  4. **DynamoDB**: Save the final `ReceiptData` record.
  5. **Response**: Return the completed `ReceiptData` JSON object to the client.

### 2. Manual Save
- **Endpoint**: `POST /api/receipts/manual`
- **Content-Type**: `multipart/form-data`
- **Body**: `metadata` (JSON String), `file` (Optional Binary)
- **Lambda Responsibility**: Save directly to DynamoDB and optionally handle S3.

### 3. GET /api/receipts
- **Responsibility**: Fetch all records from DynamoDB for the current user.

### 4. DELETE /api/receipts/:id
- **Responsibility**: Cascade delete from DynamoDB and remove the object from S3 using the stored `imageUrl`.
